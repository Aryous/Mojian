# 核心理念：Agent 优先操作原则

> 定义墨简项目作为 Agent 驱动系统的运行原则。
> 所有 Agent 和人类参与者在决策时以此为基准。

---

## 系统目标

**产品目标**：构建一个以中国古风为设计语言的 AI 驱动简历编辑器。

**系统目标**：使 Agent 能够自主、可靠地构建和维护产品，人类只在意图输入和裁决节点介入。

---

## 核心理念

### 1. Agent 看不到的知识不存在

所有决策、约束、上下文必须版本化存储于代码仓库。对话中的口头共识、
头脑中的隐性知识、外部文档中的信息——对 Agent 来说都不存在。

### 2. 用约束赋能，而非用指令微管理

通过不变量（lint 规则、结构测试、分层约束）定义"什么不能做"，
在边界内给 Agent 充分自由。强制执行边界，而非规定实现方式。

### 3. 环境先于代码

lint 规则、CI 流水线、结构测试是基础设施，不是 Agent 任务。
它们在第一行业务代码之前就应存在，并对每次提交自动运行。

### 4. 文档不足时，将规则升级为代码

文档（软约束）被反复违反 → 写入 .claude/rules/（中约束）。
.claude/rules/ 被反复违反 → 编码为 lint 规则（硬约束）。
约束只升级，不降级。

### 5. 系统需要闭合的反馈循环

开环系统无法自我纠正。每一条信息流都需要回路：
- Agent 产出 → 人类审批 → 裁决回注
- lint 报错 → Agent 修复 → 代码合规
- 文档漂移 → doc-gardening 扫描 → 修复
- Agent 失败 → 识别环境缺陷 → 补建能力

### 6. 人类掌舵，Agent 执行

人类提供意图、审批输出、裁决歧义。
Agent 负责分析、实现、验证。
两者通过协议连接，不通过临时对话。
